monoid

Monoid is one of the most important interfaces in algebra.

The basic definition can be generalized away from Set, to the notion of a monoid object in any monoidal category. Then the source category of the “composition” operation is no longer Cartesian product but a general tensor product.

Definition

Definition. A monoid is a semigroup that has an identity element \(e\) for its binary operation:

$$ e + x = x + e = x $$

Theorem. The identity element is unique.

proof: Assume there are two identity elements \(e_1, e_2\), and consider \(e_1 + e_2\). By the equation for identities, the result must be both \(e_1\) and \(e_2\). Therefore \(e_1 = e_2\).

As a single-object category

A monoid can also be regarded as a category with a single element. After all the category laws are essentially the monoid laws but for the fact that categories operations are like partial functions, i.e. arrows exist only for certain souce/target object combinations.

Its worth keeping in mind though that monoid is often a useful concept in contexts where we care about the extra structure carried by the underlying set, whereas the categorical description “doesn’t see” or care about anything except the arrows, i.e. the multiplication table.

Preparation for generalization

We need to uncurry the composition operation, and express the monoid interface in terms of the monoidal category operations: tensor product and unit.

$$ \mu : M \times M \to M $$

$$ \eta : I \to M $$

Diagramatics

The diagrams defining a monoid are precisely those defining a monoid object with the tensor product specialized to Set’s Cartesian product.

In programming

Many basic operations and data structures of interest in programming are monoids, like strings, lists, booleans, orderings, etc.

Definition in Haskell

Here is a Haskell interface definition, ever so slightly tweaked from the version in the standard library. 

class Semigroup a where
  (<>) :: a -> a -> a

class Semigroup a => Monoid a where
  identity :: a
